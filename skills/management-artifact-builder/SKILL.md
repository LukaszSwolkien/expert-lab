# Management Artifact Builder

## Overview

Use this skill when the user's next step requires a shareable artifact, not just advice. Produce practical, context-specific documents that can be used immediately.

## Workflow

### Step 1: Select Artifact Type

Choose the smallest useful artifact:

| Artifact | Use when |
|---|---|
| OKRs / goals | Aligning outcomes, metrics, ownership, and scope |
| Team charter | Defining mission, ownership, interfaces, and operating norms |
| Ownership matrix / RACI | Clarifying who decides, owns, contributes, and is informed |
| Decision record | Capturing a decision, options, rationale, and consequences |
| Hiring scorecard | Defining role expectations and interview evaluation criteria |
| Interview plan | Structuring interview loops and signal collection |
| Retrospective format | Running a focused retro with clear output |
| Stakeholder plan | Managing communication, influence, and decision flow |

Use `survey-builder` for feedback surveys. Use `one-on-one` for 1:1 agendas or feedback delivery. Use `org-design-review` before creating an org artifact when the structure itself is still undecided.

### Step 2: Confirm The Job

Before producing the artifact, identify:

- Audience.
- Decision or behavior the artifact should enable.
- Time horizon.
- Owner.
- Required level of formality.
- What must be explicit to avoid confusion.

Ask one focused question only if a missing input would materially change the artifact.

### Step 3: Build The Artifact

Use these default structures unless the user provides a preferred format:

- **OKRs:** Objective, key results, metric source, owner, scope, non-goals, review cadence.
- **Team charter:** Mission, responsibilities, boundaries, interfaces, decision rights, operating norms, escalation path.
- **RACI:** Workstream, accountable owner, responsible contributors, consulted parties, informed parties, decision point.
- **Decision record:** Context, decision, options considered, rationale, trade-offs, consequences, review date.
- **Hiring scorecard:** Role outcomes, must-have skills, nice-to-have skills, interview signals, anti-signals, calibration notes.
- **Retro format:** Goal, prompts, timeboxes, facilitation notes, decision/action capture, follow-up owner.
- **Stakeholder plan:** Stakeholders, interest, influence, concerns, message, cadence, owner.

### Step 4: Critique Before Delivery

Before finalizing, check:

- Does this artifact make ownership clearer?
- Does it expose trade-offs instead of hiding them?
- Is it specific enough to drive behavior?
- Is it lighter than the problem requires, or heavier than useful?
- Could someone use it without another explanatory meeting?

## Output Format

Return:

1. **Artifact** - ready to use, not a generic template.
2. **Usage Notes** - how to introduce or apply it.
3. **Risks / Watch-outs** - what could be misread or fail.

For artifact reviews, lead with findings and then provide a revised version if useful.

## Validation Checklist

- [ ] Artifact type matches the user's next action.
- [ ] Audience and owner are explicit.
- [ ] Trade-offs, non-goals, or boundaries are visible where relevant.
- [ ] The artifact is ready to use, not a fill-in skeleton.
- [ ] The artifact avoids unnecessary process weight.
